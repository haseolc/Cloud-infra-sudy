# ☁️ AWS & 클라우드 핵심 용어 정리

## 1. 시스템 운영 및 설계 (Reliability & Scalability)

- **고가용성 (High Availability, HA):** 시스템이 중단 없이 장기간 가동되도록 보장하는 성질. 장애 시 자동 복구를 통해 다운타임을 최소화함.
- **내결함성 (Fault Tolerance):** 시스템 구성 요소 일부에 장애가 발생해도 서비스가 중단되지 않고 유지되는 능력. (우회 처리 등을 통해 무중단 유지)
- **재해 복구 (Disaster Recovery, DR):** 자연재해나 대규모 장애 발생 시 시스템을 원래 상태로 복구하는 프로세스.
- **장애 허용 (Fault Tolerance):** 고가용성보다 높은 단계로, 장애가 발생해도 사용자에게 전혀 영향 없이 서비스가 지속되는 상태.
- **확장성 (Scalability):** 수요 증가에 맞춰 리소스 규모를 쉽고 빠르게 늘릴 수 있는 능력.
- **탄력성 (Elasticity):** 수요 변화에 따라 컴퓨팅 자원을 자동으로 확장(Scale-out)하거나 축소(Scale-in)하여 비용을 최적화하는 능력.

## 2. 클라우드 기본 개념

- **온프레미스 (On-premise):** 기업이 자체적으로 데이터 센터(전산실)를 보유하고 하드웨어를 직접 설치·운영하는 방식.
- **오프프레미스 (Off-premise):** 클라우드 환경과 같이 외부 인프라를 이용하는 방식.
- **온디멘드 (On-Demand):** 서비스가 필요할 때 즉시 요청하여 사용하는 방식.
- **프로비저닝 (Provisioning):** 사용자 요구에 맞게 IT 자원(CPU, 메모리, 저장공간 등)을 할당하고 배치하여 즉시 사용 가능한 상태로 만드는 것.

## 3. 컴퓨팅 및 배포 리소스

- **인스턴스 (Instance):** 클라우드에서 실행되는 가상 서버 (예: AWS EC2).
- **AMI (Amazon Machine Image):** 인스턴스를 실행하는 데 필요한 OS, 애플리케이션 등이 설정된 템플릿.
- **ARN (Amazon Resource Name):** AWS 내 모든 리소스를 고유하게 식별하기 위한 식별자.
- **오토 스케일링 (Auto Scaling):** 설정된 정책에 따라 서버 인스턴스 수를 자동으로 조절하는 서비스.
- **컨테이너 (Container):** 애플리케이션과 그 실행에 필요한 모든 라이브러리를 하나로 패키징한 소프트웨어 단위.
- **보안 그룹 (Security Group):** 인스턴스에 대한 가상 방화벽 (인바운드/아웃바운드 트래픽 제어).

## 4. 네트워크 및 트래픽 (Networking)

- **트래픽 (Traffic):** 네트워크를 통해 송수신되는 데이터의 양.
    - **인바운드:** 외부에서 서버로 들어오는 트래픽.
    - **아웃바운드:** 서버에서 외부로 나가는 트래픽.
- **로드 밸런서 (Load Balancer):** 부하 분산 장치. 여러 서버에 트래픽을 균등하게 분배함.
- **DNS (Domain Name System):** 도메인 주소(naver.com)를 IP 주소(123.12.34.5)로 변환해주는 시스템.
- **VPC 관련 IP:**
    - **Public IP:** 인터넷을 통해 외부와 통신 가능한 고유 주소.
    - **Private IP:** 내부 네트워크 안에서만 식별되는 주소.
    - **VIP (Virtual IP):** 특정 물리적 장비에 종속되지 않는 가상의 IP.
- **CDN (Contents Delivery Network):** 사용자 근처의 서버(Edge Location)에 콘텐츠를 캐싱하여 빠르게 전달하는 기술.
- **VPN (Virtual Private Network):** 공용 인터넷망을 사설망처럼 안전하게 사용할 수 있게 해주는 가상 터널.
- **NAT (Network Address Translation):** 사설 IP를 공인 IP로 변환하여 통신할 수 있게 하는 기술.

## 5. 스토리지 및 데이터베이스 (Storage & DB)

- **EBS (Elastic Block Store):** EC2 인스턴스에서 사용하는 블록 수준의 가상 하드디스크.
- **오브젝트 스토리지 (Object Storage):** 계층 구조 없이 데이터(파일+메타데이터)를 객체 단위로 저장하는 방식 (예: AWS S3).
- **스냅샷 (Snapshot):** 특정 시점의 데이터 상태를 그대로 복사하여 보관한 이미지.
- **SQL (Relational DB):** 표 형태의 구조화된 데이터를 관리하는 언어.
- **NoSQL (Non-Relational DB):** 정형화되지 않은 데이터를 저장하며 수평적 확장이 용이한 DB.
- **데이터 웨어하우스 (Data Warehouse):** 분석과 의사결정을 목적으로 여러 곳의 데이터를 통합 관리하는 중앙 저장소.
- **트랜잭션 (Transaction):** 쪼개질 수 없는 하나의 논리적 작업 단위 (전부 성공하거나, 전부 실패해야 함).

## 6. 보안 및 기타 (Security & Misc)

- **인증 (Authentication):** "당신은 누구인가?"를 확인하는 과정 (ID/PW 등).
- **인가 (Authorization):** 인증된 사용자에게 특정 자원에 대한 "권한"을 부여하는 것.
- **암호화/복호화:** 데이터를 알아볼 수 없는 코드로 바꾸거나(암호화), 다시 원래대로 돌리는 것(복호화).
- **SSO (Single Sign On):** 한 번의 로그인으로 연동된 여러 시스템을 이용할 수 있는 통합 인증 방식.
- **API (Application Programming Interface):** 소프트웨어 간에 데이터를 주고받기 위한 규약/인터페이스.
- **CLI (Command Line Interface):** 텍스트 터미널을 통해 명령어를 입력하여 시스템을 제어하는 방식.
