# 3️⃣ Linux 서버 운영 기초 (Ubuntu)

## 1️⃣ 디렉토리 구조

(`/etc`, `/var`, `/home`)

---

### 🗄️ 비유: 회사 건물의 층별 역할

| 디렉토리 | 비유 | 역할 |
| --- | --- | --- |
| `/etc` | 운영 규칙 문서실 | 설정 파일 |
| `/var` | 물류 창고 | 로그·데이터 |
| `/home` | 개인 사무실 | 사용자 파일 |

---

### 💻 기술적 설명

### `/etc`

- 시스템 설정 파일
- nginx, ssh 설정

### `/var`

- 로그 (`/var/log`)
- DB 데이터
- 캐시

### `/home`

- 사용자 홈
- 작업 파일

---

### 🔍 이 프로젝트에서

- 백업 대상:
    - `/var/www`
    - `/var/lib/mysql`
- 보안 설정:
    - `/etc/nginx`
    - `/etc/ssh`

👉 **백업 범위 판단의 기준**

---

## 2️⃣ 사용자 / 권한

---

### 🔐 비유: 출입증과 직급

- 누구인가?
- 무엇을 할 수 있는가?

---

### 💻 개념

> Linux는 사용자 + 그룹 + 권한으로 모든 접근을 통제
> 

---

### 📄 권한 구조

```
rwxr-x---

```

- 읽기(r)
- 쓰기(w)
- 실행(x)

---

### 🔍 이 프로젝트에서

- root 최소 사용
- 서비스 계정 분리
- DB 접근 권한 제한

👉 **침해 사고 최소화**

---

## 3️⃣ 프로세스 개념

---

### 🧠 비유: 실제로 일하는 직원

- 프로그램 = 직원 명단
- 프로세스 = 지금 일 중인 직원

---

### 💻 기술적 정의

> 프로세스 = 실행 중인 프로그램 인스턴스
> 

---

### 📌 예시

- nginx 실행 → 여러 worker 프로세스
- mysql 데몬

---

### 🔍 이 프로젝트에서

- 죽은 프로세스 확인
- 장애 원인 분석
- Kubernetes의 Pod 개념과 연결

👉 **“서버가 살아 있는지” 판단 기준**

---

## 4️⃣ 포트 확인

---

### 🚪 비유: 문이 열려 있는지 확인

- 서버 주소 = 건물
- 포트 = 문 번호

---

### 💻 개념

> 포트는 프로세스가 외부와 통신하는 통로
> 

---

### 📌 확인 명령

```bash
ss -tulnp

```

---

### 🔍 이 프로젝트에서

- 80 / 443 → Nginx
- 내부 포트는 비공개

👉 **보안 점검 필수 단계**

---

## 5️⃣ 서비스 실행 방식

---

### 🔁 비유: 자동 출근 vs 수동 출근

---

### 🧑‍🔧 수동 실행

```bash
nginx

```

- 재부팅 시 꺼짐

---

### 🤖 서비스 등록

```bash
systemctlenable nginx

```

- 부팅 시 자동 실행

---

### 💻 개념

> Linux 서비스는 systemd가 관리
> 

---

### 🔍 이 프로젝트에서

- 서버 재부팅에도 서비스 유지
- 장애 복구 자동화

👉 **운영 서버의 기본 조건**

---

## 6️⃣ 로그 개념

---

### 📜 비유: CCTV + 업무일지

- 무슨 일이 있었는지 기록
- 문제 발생 시 원인 추적

---

### 💻 로그란?

> 시스템과 서비스의 행동 기록
> 

---

### 📂 주요 로그 위치

- `/var/log/syslog`
- `/var/log/nginx/access.log`
- `/var/log/nginx/error.log`

---

### 🔍 이 프로젝트에서

- 접속 기록 분석
- 공격 탐지
- 장애 원인 분석

👉 **운영자의 눈**

---

# 🧠 3단계 핵심 요약

- 디렉토리 구조는 서버의 역할 분리
- 사용자와 권한은 보안의 기본
- 프로세스는 실행 중인 서비스
- 포트는 외부와의 연결 창구
- 서비스는 자동 실행되어야 운영 가능
- 로그는 장애와 공격을 추적하는 근거
