# 1️⃣-1️⃣ 클라우드 기본

## 1️⃣ 온프레미스(On-Premise) vs 클라우드(Cloud)

---

### 🏢 비유: 회사 건물 vs 공유 오피스

### 🧱 온프레미스 = 자기 건물

- 건물 직접 구매
- 전기·냉난방·보안·청소 전부 직접 관리
- 고장 나면 직접 수리

### ☁️ 클라우드 = 공유 오피스

- 필요할 때 빌려 씀
- 전기·인터넷·보안 기본 제공
- 방 크기 늘리고 줄이기 자유

---

### 💻 IT 관점 비교

| 구분 | 온프레미스 | 클라우드 |
| --- | --- | --- |
| 초기 비용 | 매우 큼 | 거의 없음 |
| 확장 | 어려움 | 즉시 가능 |
| 관리 | 직접 | 서비스 제공자 |
| 장애 대응 | 직접 | 자동화 가능 |

---

### 🔍 이 프로젝트에서

- **AWS EC2 사용** → 클라우드
- 서버를 코드로 생성하고 삭제 가능
- 실습·공유·백업에 최적

👉 **팀 프로젝트 + 자동화**에는 클라우드가 정답

---

## 2️⃣ IaaS / PaaS / SaaS

---

### 🍱 비유: 요리 단계별 서비스

---

### 🧱 IaaS (Infrastructure as a Service)

### 비유: **주방만 빌려줌**

- 냄비, 가스, 싱크대 제공
- 요리는 내가 함

**예시**

- AWS EC2
- Azure VM

👉 OS 설치부터 전부 직접 관리

---

### 🍳 PaaS (Platform as a Service)

### 비유: **요리 도구 + 기본 재료 제공**

- 불, 냄비, 양념 준비됨
- 요리만 하면 됨

**예시**

- AWS Elastic Beanstalk
- Google App Engine

👉 인프라 신경 덜 씀

---

### 🥡 SaaS (Software as a Service)

### 비유: **완성된 음식 배달**

- 그냥 먹기만 하면 됨

**예시**

- Gmail
- Notion
- Google Drive

---

### 🧠 한눈에 정리

| 구분 | 내가 관리하는 것 |
| --- | --- |
| IaaS | OS, 앱, 보안 |
| PaaS | 앱 코드 |
| SaaS | 거의 없음 |

---

### 🔍 이 프로젝트에서

- **IaaS 선택**
- 이유:
    - Terraform
    - Docker
    - Kubernetes
    - 보안·백업 직접 설계

👉 **“운영 공부” 목적에 최적**

---

## 3️⃣ 리전(Region) / 가용 영역(AZ)

---

### 🌍 비유: 나라와 도시

### 🌎 Region

- 물리적으로 떨어진 지역
- 예: 서울, 도쿄, 버지니아

### 🏙️ AZ (Availability Zone)

- 같은 리전 안의 독립된 데이터센터
- 전기·네트워크 분리

---

### 🏗️ 구조 그림 (개념)

```
서울 리전
 ├─ AZ-a (데이터센터1)
 ├─ AZ-b (데이터센터2)
 └─ AZ-c (데이터센터3)

```

---

### ⚡ 왜 나누는가?

- 한 AZ 장애 → 다른 AZ는 정상
- **고가용성(HA)** 확보

---

### 🔍 이 프로젝트에서

- 단일 AZ → 기본 구조
- 확장 시:
    - Load Balancer
    - Multi-AZ

👉 **장애 대비 개념 설명에 매우 좋음**

---

## 4️⃣ VM이란 무엇인가?

---

### 🖥️ 비유: 아파트 안의 독립된 방

- 건물 = 물리 서버
- 방 = VM
- 각 방마다:
    - 문 있음
    - 전기·가구 따로
    - 서로 간섭 없음

---

### 💻 기술적 정의

> VM = 하나의 물리 서버를 가상으로 나눈 독립된 컴퓨터
> 
- 각 VM은:
    - 자체 OS
    - CPU / RAM 할당
    - 네트워크 독립

---

### ⚙️ 하이퍼바이저

- VM을 관리하는 관리자
- ESXi, KVM, Hyper-V 등

---

### 🔍 이 프로젝트에서

- EC2 = VM
- Ubuntu 설치
- 그 위에:
    - Docker
    - Kubernetes

👉 **모든 자동화의 출발점**

---

# 🧠 1-1단계 핵심 요약

- 클라우드는 서버를 빌려 쓰는 방식
- IaaS는 인프라부터 직접 운영
- 리전/AZ는 장애 대비를 위한 물리적 분리
- VM은 클라우드에서 제공하는 기본 서버 단위
