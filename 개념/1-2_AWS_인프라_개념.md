# 1️⃣-2️⃣ AWS 인프라 개념

## 1️⃣ EC2 (Elastic Compute Cloud)

---

### 🖥️ 비유: 임대 가능한 개인 컴퓨터

- AWS가 가진 거대한 컴퓨터 창고
- 그중 한 대를 **필요한 시간만 빌려 쓰는 것**

---

### 💻 기술적 정의

> EC2 = 클라우드에서 제공하는 가상 서버(VM)
> 
- OS 선택 가능 (Ubuntu, Amazon Linux 등)
- CPU, RAM 크기 선택
- 켜고/끄고/삭제 가능

---

### ⚡ 왜 “Elastic”?

- 서버 크기 변경 가능
- 서버 수 늘렸다 줄였다 가능

---

### 🔍 이 프로젝트에서

- EC2 = **영구 서버**
- Ubuntu 설치
- Docker / Kubernetes 실행

👉 **모든 서비스의 기반**

---

## 2️⃣ 보안 그룹 (Security Group)

---

### 🚪 비유: 건물 출입 규칙표

- “누가”
- “어떤 문으로”
- “들어올 수 있는지”

를 정해놓은 규칙

---

### 🔐 기술적 정의

> Security Group = 인스턴스 단위 방화벽
> 
- 허용된 트래픽만 통과
- 기본은 전부 차단

---

### 📋 예시 규칙

| 규칙 | 의미 |
| --- | --- |
| TCP 22 허용 | SSH 접속 |
| TCP 80 허용 | HTTP |
| TCP 443 허용 | HTTPS |

---

### 🔍 이 프로젝트에서

- 외부:
    - 80 / 443 → Nginx Ingress
- 내부:
    - 컨테이너 포트 숨김

👉 **보안의 1차 방어선**

---

## 3️⃣ 키 페어 (Key Pair)

---

### 🔑 비유: 열쇠와 자물쇠

- 서버 문 = EC2
- 자물쇠 = 공개키
- 내 열쇠 = 개인키 (.pem)

---

### 💻 기술적 정의

> Key Pair = EC2에 SSH 접속하기 위한 인증 수단
> 
- 비밀번호 로그인 ❌
- 키 기반 인증 ✅

---

### ⚠️ 중요 포인트

- 개인키는 다시 못 받음
- 분실 = 서버 접속 불가

---

### 🔍 이 프로젝트에서

- 팀원 공유 주의
- 백업 필수
- CI/CD 서버 접근 시 사용

---

## 4️⃣ 퍼블릭 IP / 프라이빗 IP

---

### 🌐 비유: 집 주소 vs 집 안 방 번호

---

### 🌍 퍼블릭 IP

- 인터넷에서 보이는 주소
- 외부 접속용

👉 예: `3.xxx.xxx.xxx`

---

### 🏠 프라이빗 IP

- 내부 네트워크용 주소
- 외부에서는 접근 불가

👉 예: `172.31.x.x`

---

### 🧠 차이 요약

| 구분 | 퍼블릭 IP | 프라이빗 IP |
| --- | --- | --- |
| 외부 접근 | 가능 | 불가능 |
| 용도 | 서비스 노출 | 내부 통신 |
| 보안 | 위험 | 안전 |

---

### 🔍 이 프로젝트에서

- 외부:
    - 퍼블릭 IP → Nginx
- 내부:
    - 프라이빗 IP → 컨테이너 통신

👉 **외부 노출 최소화 구조**

---

## 5️⃣ 포트 개념 (80, 443 등)

---

### 🚪 비유: 건물 안의 여러 출입문

- 주소(IP) = 건물
- 포트 = 문 번호

---

### 🔢 대표 포트

| 포트 | 용도 |
| --- | --- |
| 22 | SSH |
| 80 | HTTP |
| 443 | HTTPS |
| 3306 | MySQL |

---

### 💥 포트 충돌이란?

- 한 포트에는 하나의 서비스만 가능
- 여러 서비스가 80번 쓰면 충돌

---

### 🔍 이 프로젝트에서

- 외부:
    - 80 / 443 → Ingress
- 내부:
    - 각 컨테이너 개별 포트

👉 **Reverse Proxy 구조의 이유**

---

# 🧠 1-2단계 핵심 요약

- EC2는 클라우드 VM
- Security Group은 인스턴스 방화벽
- Key Pair는 서버 접근 열쇠
- 퍼블릭 IP는 외부, 프라이빗 IP는 내부
- 포트는 서비스의 출입문
