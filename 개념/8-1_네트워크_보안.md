## 0️⃣ 한 줄 요약 (큰 그림)

> 외부에서는 하나의 문만 보이게 하고,
안에서는 여러 서비스가 안전하게 움직이게 하는 구조
> 

---

## 1️⃣ Reverse Proxy

---

### 🚪 비유: 건물 입구 안내 데스크

- 외부 손님은 **입구 하나만** 본다
- 안내원이 내부 사무실로 연결

---

### 💻 기술적 정의

> Reverse Proxy = 외부 요청을 받아 내부 서버로 대신 전달하는 중개자
> 

---

### 🔁 흐름

```
사용자 → Reverse Proxy → 내부 서비스

```

---

### 📌 역할

- 실제 서버 주소 숨김
- 요청 검사
- 트래픽 분배

---

### 🧠 Ingress와의 관계

- Ingress Controller가 **Reverse Proxy 역할** 수행
- (보통 Nginx)

---

---

### 🏢 비유: 건물 출입구 하나만 열어두기

- 문이 많을수록 위험
- 관리 포인트 증가

## 8️⃣ 보안 개념 (Ingress 중심)

### 8-1. 네트워크 보안

> 🔵 핵심 요약 (큰 그림)
> 
> 
> 외부에서는 하나의 안전한 입구만 열어두고,
> 
> 내부 서비스들은 직접 노출되지 않도록 숨기는 구조
> 

---

## 1️⃣ Reverse Proxy

### 🧠 비유

> 건물 입구의 안내 데스크
> 
- 방문자는 **안내 데스크만** 만난다
- 실제 사무실 위치는 모른다
- 안내원이 대신 전달해준다

---

### 💻 기술적 정의

**Reverse Proxy란?**

> 외부 요청을 직접 서버가 받지 않고
> 
> 
> **중간에서 대신 받아 내부 서버로 전달하는 중개 서버**
> 

---

### 🔁 요청 흐름

```
사용자
  ↓
Reverse Proxy (Ingress Controller)
  ↓
내부 서비스 (Pod / Service)

```

---

### ✅ 하는 역할

- 내부 서버 주소 숨김
- 요청 검사 및 필터링
- 트래픽 분배
- SSL 종료 지점

---

## 2️⃣ 단일 진입점 (Single Entry Point)

### 🧠 비유

> 건물 출입문을 하나만 열어두는 것
> 
- 출입문이 많을수록 관리 어려움
- 보안 사고 가능성 증가

---

### 💻 기술적 정의

> 외부에서 접근 가능한 유일한 네트워크 입구를 하나로 제한하는 구조
> 

---

### 📌 장점

- 방화벽 설정 단순화
- 모니터링 지점 명확
- 보안 정책 일괄 적용 가능

---

### 🧠 Kubernetes에서

- Ingress Controller만 외부 노출
- 나머지 서비스는 Cluster 내부 전용

---

## 3️⃣ 포트 은닉 (Port Hiding)

### 🧠 비유

> 방 번호를 외부에 공개하지 않는 사무실
> 
- 손님은 안내 데스크만 안다
- 내부 방 번호는 모른다

---

### 💻 기술적 정의

> 외부에 80 / 443 같은 최소 포트만 공개하고
> 
> 
> 내부 서비스 포트는 외부에서 직접 접근 불가하게 만드는 것
> 

---

### 🔍 예시

| 구분 | 포트 |
| --- | --- |
| 외부 노출 | 80, 443 |
| 내부 서비스 | 8080, 3306, 5000 등 |

---

### 🛡 효과

- 포트 스캐닝 방어
- 직접 공격 차단
- 공격 표면 최소화

---

## 🔗 Ingress와 네트워크 보안의 관계 정리

```
[ 외부 사용자 ]
        ↓ (80 /443)
[ Ingress Controller ]
        ↓
[ 내부 Service ]
        ↓
[ Pod ]

```

---

## ✅ 8-1 핵심 정리

- Reverse Proxy: 외부 요청을 대신 받는 중개자
- 단일 진입점: 외부에 노출되는 입구는 하나
- 포트 은닉: 내부 포트는 외부에서 보이지 않게
