💻 EC2 (Amazon Elastic Compute Cloud) 핵심 정리
1. 개요
정의: EC2는 AWS에서 독립된 컴퓨터를 임대해주는 클라우드 컴퓨팅 서비스입니다.

비유: PC방에서 요금을 내고 컴퓨터를 이용하는 것처럼, AWS에서 가상 컴퓨터를 대여하여 사용하는 것과 같습니다.

2. EC2 특징 요약
유연한 확장: 컴퓨팅 요구사항 변화에 따라 파워를 조정할 수 있습니다.

비용 절감: 실제로 사용한 용량만큼만 지불합니다.

OS 선택: Linux 또는 Windows 중 원하는 환경을 쉽게 선택할 수 있습니다.

빠른 구축: 몇 분이면 전 세계에 수백 대의 컴퓨터를 생성 가능합니다.

범용성: 머신러닝, 웹 서버, 게임 서버 등 용도에 최적화된 구성이 쉽습니다.

연동성: 다른 AWS 서비스와 유기적으로 연동됩니다.

3. EC2의 하드웨어 구성 대입
일반적인 서버 구성을 EC2 서비스에 대입하면 다음과 같습니다.

컴퓨팅 (CPU/RAM): 인스턴스 (Instance)

하드디스크: EBS (Elastic Block Storage)

랜카드: ENI (Elastic Network Interface)

② EBS (Elastic Block Storage)
역할: 인스턴스가 연산(CPU/메모리)을 하는 동안 데이터를 저장하는 가상 하드디스크입니다.

특징: 영구 블록 스토리지 볼륨을 제공하며, 사용량에 따라 빠르게 확장 가능하고 프로비저닝한 만큼만 비용을 지불합니다.

③ AMI (Amazon Machine Image)
정의: 운영체제(OS)와 소프트웨어 구성이 담긴 템플릿입니다.

용도: EC2 인스턴스를 실행하기 위한 정보를 모아둔 단위입니다.

4. EC2 구축 세팅 & 사용
(이미지: 관리 콘솔 및 API를 통한 EC2 생성 및 SSH 원격 접속 과정)

인스턴스 생성: AWS 관리 콘솔, CLI, API 등을 통해 가상 머신을 생성 및 관리합니다.

이미지 선택: Amazon S3 등에 저장된 AMI(윈도우, 리눅스 등)를 선택하여 인스턴스를 구동합니다.

스토리지 연결: 인스턴스에 Amazon EBS 가상 블록 디스크를 연결하여 사용합니다.

원격 접속: SSH 공개키 인증 등을 통해 외부에서 생성된 서버에 접속하여 작업을 수행합니다.
